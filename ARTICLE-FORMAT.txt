============================================================
THE HALLUCINATED LAB — ARTICLE FORMAT SPECIFICATION
(prompt / build guide for creating a new article page)
============================================================
Use this as the instruction set when creating any new article
for the Media page. The goal: every article FEELS like the
site (same text design, colors, typography) while being free
to invent its own structure, presentation, and interactive
features. An article is a self-contained HTML file placed in
the /articles/ directory.

------------------------------------------------------------
1. THE CONTRACT — WHAT MUST STAY THE SAME
------------------------------------------------------------
These are non-negotiable so articles feel native to the site.
(Full token values in DESIGN-LANGUAGE.txt.)

a) Load the site styles first, then your own:
     <link rel="stylesheet" href="[Google Fonts Outfit+JetBrains Mono]">
     <link rel="stylesheet" href="../styles.css">
     <link rel="stylesheet" href="article.css">
     <style> /* article-specific styles only */ </style>

b) Typography rules
   - Headings: 'Outfit' (700-800 titles, 600 sub-heads).
   - Body text: 'JetBrains Mono', 0.88-0.95rem, line-height
     1.7-1.8, color var(--text-secondary).
   - Hero/article title uses the gold gradient text fill.
   - h3 color var(--gold-light); strong -> var(--text-primary);
     links -> gold with soft underline; inline code -> gold
     tinted chip. Never introduce new fonts.

c) Color rules
   - Page background var(--bg-primary) #050505; panels
     var(--bg-card); all accents from the gold family.
   - New feature colors (e.g. for a game or chart) must be
     desaturated/dark-friendly and used sparingly — gold stays
     the dominant accent.

d) Site chrome
   - Standard fixed navbar (Home / Utilities / Assistant /
     Media / Solutions / Contact, Media marked active) with
     paths prefixed ../ , the skip-link, and the standard
     footer line.
   - Reading progress bar (3px gold gradient, fixed top).
   - A "Back to Media" link somewhere obvious.

e) Head block
   - <title>Article Title - The Hallucinated Lab</title>
   - meta description, og:title/description/type=article,
     canonical https://thehallucinatedlab.space/articles/<file>,
     favicon ../assets/images/logo.jpeg.

f) Accessibility
   - Semantic landmarks, alt text, aria-labels on icon
     controls, keyboard-reachable interactive elements,
     gold :focus-visible outline (inherited from styles.css).

------------------------------------------------------------
2. THE FREEDOM — WHAT EACH ARTICLE MAY INVENT
------------------------------------------------------------
- Overall structure: an article does NOT have to follow the
  hero + sidebar TOC + linear-prose layout of
  sample-article.html. Tabs, chapters, scroll-driven scenes,
  split panels, full-bleed sections, step-by-step wizards —
  all allowed.
- Presentation style: charts, timelines, terminals,
  simulations, annotated diagrams, card decks, quizzes.
- Custom CSS/JS: keep it inline (<style>/<script>) or scoped
  to the article file so it can never leak into other pages.
  Prefix custom classes if there is any collision risk.
- Interactivity engine: vanilla JS preferred. GSAP +
  ScrollTrigger (CDN) is the approved animation library:
     gsap.min.js, ScrollTrigger.min.js, ScrollToPlugin.min.js
  ALWAYS degrade gracefully if a CDN script fails (the site
  pattern: if GSAP is missing, reveal everything statically).

------------------------------------------------------------
3. READY-MADE BUILDING BLOCKS (article.css, already loaded)
------------------------------------------------------------
Use these before inventing new ones:

  .reading-progress      fixed top progress bar (wire via JS)
  .article-hero          parallax hero: bg layer + dark
                         overlay + category badge + read time
                         + gradient title + author row
  .article-toc           sticky sidebar table of contents with
                         scroll-spy (.toc-link.active)
  .article-content       65ch prose column with all text
                         styles (h2/h3/p/a/strong/blockquote/
                         code/pre) preconfigured
  .highlight             hover-fill gold text highlighter
  .tooltip-term          dashed-underline term with
                         data-tooltip hover card
  blockquote + cite      gold-left-bar pull quote
  pre + .code-copy-btn   code block with copy-to-clipboard
  .interactive-box       hover-lift card for callouts/demos
  .deep-dive             <details> expandable "go deeper" box
  .article-figure        bordered figure + caption (images)
  .article-divider       thin rule with centered gold diamond
  .article-footer-tags   pill tag row
  .article-back-link     pill "back to media" button
  .gsap-reveal           marks an element for scroll-reveal
                         (article.js animates it)

article.js (optional include) provides: progress bar scrub,
hero parallax, scroll reveals, TOC scroll-spy + smooth scroll,
copy buttons. Include it only if you use those patterns;
otherwise ship your own scoped script.

------------------------------------------------------------
4. RICH MEDIA RULES
------------------------------------------------------------
PHOTOS / IMAGES
- Wrap in .article-figure with a <figcaption> credit/caption.
- Store local images in /articles/assets/ (create if needed)
  or reuse /assets/images/. Always set width/height or
  aspect-ratio to avoid layout shift; loading="lazy" for
  below-the-fold images. Dark-friendly imagery preferred;
  if an image is bright, the 1px gold-tinted border and
  rounded corners (radius-lg) keep it on-brand.

VIDEO REFERENCES
- Never autoplay. Two approved patterns:
  1) Reference card: an .interactive-box with a play-icon
     thumbnail linking out (YouTube etc.), title, channel,
     duration, and a one-line "why watch this".
  2) Embed: <iframe> (youtube-nocookie.com preferred) inside
     a 16:9 wrapper (aspect-ratio: 16/9; width: 100%;
     radius-lg; 1px --border-subtle border), with title
     attribute set, ideally click-to-load (thumbnail first,
     inject iframe on click) to keep the page fast.

GAMES / PLAYGROUNDS / INTERACTIVE UI
- Each article may ship its own interactive widget: a mini
  game, simulator, calculator, visualizer, quiz, sandbox.
- Container: a panel styled like the site (--bg-card,
  1px --border-subtle, radius-lg, padding 1.5-2rem), with a
  small UPPERCASE gold label header and, if useful, a status
  chip with the pulse-dot animation.
- Controls (buttons, sliders, inputs) follow the site form
  language: dark inputs, gold accents, pill buttons,
  accent-color: var(--gold-primary) for range/radio.
- Must be keyboard operable, must not capture page scroll,
  must run entirely client-side (no backend, no tracking),
  and must fit mobile widths (stack below 700px).
- State may persist in localStorage using the "thl_" key
  prefix (site convention: e.g. thl_community_posts).
- Heavy logic stays inside the article file; do not modify
  shared site JS/CSS for one article.

------------------------------------------------------------
5. PUBLISHING CHECKLIST
------------------------------------------------------------
1) Save the file as /articles/<kebab-case-slug>.html.
2) Register it in media.js -> ARTICLES array:
     {
       id: '<kebab-slug>',
       title: '<Title>',
       author: 'Pratyush' | 'Divyansh Tripathi' | '...',
       date: 'YYYY-MM-DD',
       category: 'AI & ML' | 'Quantum Computing' |
                 'Open Source' | 'Privacy & Security' |
                 'Dev Tools' | 'Research' | 'General',
       excerpt: '1-2 sentence hook for the card.',
       coverImage: null,
       coverGradient: 'linear-gradient(135deg, <3 dark stops>)',
       status: 'published',        // or 'upcoming'
       featured: true|false,       // featured section slot
       articleUrl: 'articles/<kebab-slug>.html',
     }
   The coverGradient is the card art on the Media page —
   pick 3 dark stops that echo the article's own palette.
3) Test: desktop + <=700px mobile, keyboard-only pass,
   with CDN blocked (graceful degradation), and confirm the
   card renders in Featured/Archive on media.html.
4) Commit and push to main; GitHub Pages deploys the site.
